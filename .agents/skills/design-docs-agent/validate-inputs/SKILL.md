# Validate Inputs

## Purpose

Run the canonical preflight gate for the shared Design Docs package. This skill
owns the public `sourceBundle` contract, mode detection, paywall enforcement,
and the entry assertions against the external TRR-APP contract.

## Use When

1. The orchestrator enters the `validation` phase.
2. The package must fail early on malformed or underspecified input after a
   caller-supplied or auto-acquired bundle is available.

## Do Not Use For

1. Extraction work after preflight has already passed.
2. Brand-tab synchronization or verification gates.

## Inputs

- `articleUrl`
- optional `sourceBundle`
- `contracts/source-bundle.schema.json`
- `contracts/publisher-policy.yaml`
- `contracts/external-app-contract.yaml`
- `TRR-APP/apps/web/src/lib/admin/design-docs-config.ts`
- `TRR-APP/apps/web/src/lib/admin/design-docs-pipeline-types.ts`

## Procedure

1. Require `articleUrl`.
2. Require either:
   - caller-supplied `sourceBundle`, or
   - bundle generated by `fetch-source-bundle` earlier in the `validation`
     phase
3. Validate `sourceBundle` against `contracts/source-bundle.schema.json`.
4. Read `contracts/publisher-policy.yaml` and record any policy notes for the
   source host:
   - known paywalled article host
   - allowed public supporting source
   - unknown or unlabeled article host
5. Resolve one orchestration mode:
   - `add-article`
   - `add-first-article`
   - `create-brand`
   - `update-article`
6. Resolve source mode per `references/source-html-modes.md`:
   - Mode A browser-save
   - Mode B view-source
   - merged Mode A + Mode B bundle
   - rendered bundle with companion assets
7. Record rendered-source authority and component-inventory authority from the
   supplied bundle fields before extraction starts.
8. Enforce bundle-precondition policy from `contracts/publisher-policy.yaml`.
   - `fetch-source-bundle` may have attempted live acquisition first.
   - `validate-inputs` only proceeds when a valid bundle now exists.
   - If no bundle exists, return the blocking acquisition report and stop.
9. Run the external-app preflight assertions described by
   `contracts/external-app-contract.yaml`.
10. Return the resolved mode, source-mode classification, authority notes, and
   any blocking input errors.

## Rule

Do not duplicate source-bundle shape, paywall domains, or external-app contract
details here. The contract files under `contracts/` are canonical.
