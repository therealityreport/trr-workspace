How the app is built (know before you mod)

ElectronChromium + Node

app.asarBundled JS/CSS/HTML

Claude Desktop is a signed Electron app. The UI is a web app compiled into Claude.app/Contents/Resources/app.asar. Modifying the UI means unpacking that archive, injecting CSS, and repacking it. The app is code-signed, so after modification macOS Gatekeeper will warn on launch and you'll need to re-sign (ad-hoc) or remove the quarantine attribute.

Implementation plan (phased)

Phase 0 · Recon

Locate and inspect the bundle

cd /Applications/Claude.app/Contents/Resources — confirm an app.asar exists. Install npx @electron/asar. Extract to a workspace copy so the original stays untouched.

Phase 1 · Extract

Unpack app.asar and map the CSS

npx @electron/asar extract app.asar ./app-unpacked. Grep for --bg, background-color, or the hex #d97757 to find the design tokens that drive the UI.

Phase 2 · Inject

Add an override stylesheet

Create custom-theme.css and load it from the renderer HTML (usually index.html) via a <link> tag placed last so specificity wins. Keep overrides to CSS variables where possible — future-proof and reversible.

Phase 3 · Repack

Rebuild the asar and re-sign

npx @electron/asar pack ./app-unpacked app.asar. Then xattr -cr /Applications/Claude.app to drop quarantine, and codesign --force --deep --sign - /Applications/Claude.app for an ad-hoc signature.

Phase 4 · Automate

Write an idempotent installer

A shell script that: detects the Claude version, backs up the original asar, applies the patch, re-signs, and can fully revert. Re-run after every app update.

Phase 5 · Distribute (optional)

Publish as a theme pack

Ship the CSS + installer as a GitHub repo. Match the structure of existing theme projects (see below) so users can drop in new palettes without re-patching.

Patch snippet (illustrative)

# 1. Unpack
cd /Applications/Claude.app/Contents/Resources
sudo cp app.asar app.asar.backup
sudo npx @electron/asar extract app.asar /tmp/claude-unpacked

# 2. Drop in override CSS
cat > /tmp/claude-unpacked/custom-theme.css <<'CSS'
:root{
  --bg-000: #0f1419 !important;
  --bg-100: #1c2128 !important;
  --accent:  #58a6ff !important;
  --text-100:#e6edf3 !important;
}
CSS

# 3. Link it from the renderer HTML (append <link> last)
printf '<link rel="stylesheet" href="custom-theme.css">' \
  >> /tmp/claude-unpacked/index.html

# 4. Repack + re-sign
sudo npx @electron/asar pack /tmp/claude-unpacked app.asar
sudo xattr -cr /Applications/Claude.app
sudo codesign --force --deep --sign - /Applications/Claude.app
Heads up: This modifies a signed application. It will break on every auto-update, may violate Anthropic's terms for redistribution, and can prevent future updates from applying cleanly. Always keep app.asar.backup and be ready to revert.

Risks & mitigations

Update churn

Breaks on every release

Auto-update replaces app.asar. Mitigate: version-pinned installer with one-command re-apply.

Signing

Gatekeeper refuses to launch

Modifying a signed bundle invalidates the signature. Mitigate: codesign --force --deep --sign - for ad-hoc, and xattr -cr to clear quarantine.

ToS

Redistribution concerns

Ship CSS + installer only — never the patched asar or Anthropic assets. Ship diffs, not binaries.

Alternative worth trying first: check Claude Desktop's Settings for a built-in theme toggle or high-contrast mode before patching. A supported setting is always better than a modded bundle.
