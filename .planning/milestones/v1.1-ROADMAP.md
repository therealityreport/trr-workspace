# Milestone v1.1: Cloud-First / No-Docker Workspace Tooling

**Status:** ✅ SHIPPED 2026-04-04
**Phases:** 6-8
**Total Plans:** 3

## Overview

This milestone shifted the TRR workspace to a cloud-first development and validation contract. It first froze the preferred remote-first path and safety boundaries, then aligned root scripts and profiles to that contract, and finally proved the no-Docker lane in practice while documenting the remaining Docker-only fallback cases.

## Milestone Summary

**Key Decisions:**

- Treat no-Docker workspace operation as a milestone-level tooling goal, not an incidental preference
- Prefer isolated remote Supabase branches or disposable environments over local Docker when they answer the same validation question
- Keep Docker-backed paths available only as narrow, explicit fallback lanes instead of silently removing them

**Issues Resolved:**

- Mixed messaging about whether Docker was required for ordinary workspace development
- Backend schema-validation guidance that implied local Docker replay as the default
- Unclear distinction between canonical profiles and compatibility/fallback profiles

**Issues Deferred:**

- Full removal of Docker-backed fallback lanes
- Automated remote branch/disposable DB provisioning for validation

**Technical Debt Incurred:**

- Milestone execution happened inline on a dirty `main`, so historical git-range stats are artifact-derived rather than isolated from a clean feature branch
