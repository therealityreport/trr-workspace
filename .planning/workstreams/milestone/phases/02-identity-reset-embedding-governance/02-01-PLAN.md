<objective>
Move face-reference ownership and DeepFace-backed identity governance into `TRR-Backend` so approved reference images, embeddings, and stateless search/verify flows live in canonical `ml.*` state without changing the v1 ArcFace-class contract.

Purpose: Phase 1 fixed the retained asset contract. Phase 2 fixes the identity contract that later screentime execution depends on. If approval state, duplicate control, and embedding provenance stay implicit, Phase 3 cannot port execution safely.
Output: A backend-owned face-reference lane with review-state governance, donor bridge semantics, explicit embedding provenance, and admin APIs for list/review/search/verify under one ArcFace-class contract.
</objective>
