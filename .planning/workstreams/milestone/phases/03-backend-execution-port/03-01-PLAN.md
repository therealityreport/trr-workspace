<objective>
Port cast-screentime execution from the standalone `screenalytics` runtime into `TRR-Backend` without changing the retained admin route surface, while preserving reproducibility, artifact parity, and reversible cutover control.

Purpose: Phases 1 and 2 already froze the backend-owned asset and identity contracts. Phase 3 makes those contracts operational by moving the screentime executor itself behind the retained backend seam instead of relying on donor HTTP.
Output: A backend-owned executor path for run launch and clip generation, backed by retained `ml.*` state, explicit runtime-mode gating, versioned run config, targeted parity tests, and continuity docs.
</objective>
