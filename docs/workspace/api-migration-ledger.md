# API Migration Ledger

This ledger maps TRR-APP direct SQL callers to backend API routes as direct SQL
is moved behind TRR-Backend.

## Required Row Fields

| Field | Meaning |
|---|---|
| App caller | File and function that previously used direct SQL. |
| Backend route | TRR-Backend route that now owns the data access. |
| App client | TRR-APP proxy/repository client that calls the backend route. |
| Auth context | Firebase/admin/internal context preserved by the migration. |
| Cacheability | Route cache, in-flight dedupe, and invalidation behavior. |
| Error contract | User/operator-visible error shape. |
| Validation | Focused backend/app tests or manual route proof. |
